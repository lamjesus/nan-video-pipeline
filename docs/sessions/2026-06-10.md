# Sesión 2026-06-10 — Selección visual real + config + yarn + docs

Bitácora de lo cambiado en esta PR y por qué. Memoria para el equipo.

## Qué se hizo

- **Evaluación por visión rehecha (Tarea C).** `mimo-v2.5` está ciego en el cluster;
  la evaluación pasa a `gemma4` (fallback `qwen3.6`) con la imagen en **base64**
  (formato array OpenAI), y `User-Agent` en las descargas de Wikimedia. La tubería
  descarga las candidatas una vez (en memoria) y guarda la elegida sin re-descargar.
- **`config.yml`.** Modelos, voz por defecto y providers salen del código a un YAML
  versionado. `.env` queda sólo para secretos + overrides opcionales (sin valores
  duplicados entre ambos).
- **Carga de `.env` arreglada.** Faltaba `dotenv/config`: el `.env` no se leía y
  `yarn doctor` fallaba aunque estuviera configurado. Ahora se carga en `config` y
  en `doctor`.
- **npm → yarn** en todo el repo (CI, scripts, docs). Lockfile: `yarn.lock`;
  `package-lock.json` fuera.
- **Lógica pura extraída + tests.** `src/pipeline/vision-util.ts` (search terms,
  ext/mime, ranking) con 14 tests nuevos → **30 tests** en total.
- **`docs/` reorganizado.** `TAREAS.md` movido aquí; nuevos `TROUBLESHOOTING.md`,
  `caso-uso-1.md` y este `sessions/`. README y AGENTS apuntan a `docs/`.
- **Limpieza de PR.** `.gitignore` para `package-lock.json` y carpetas de
  herramientas de IA; referencias colgantes a `PROGRESS.md` redirigidas.

## Verificación

- `yarn typecheck` ✅ · `yarn test` ✅ 30/30 · `yarn doctor` ✅ (cluster lista
  `gemma4`).
- **E2E real contra el cluster:** `yarn vision caso-ejemplo` → 9/9 imágenes;
  `gemma4` acierta en la mayoría (volcán, ruinas). Detalle en `caso-uso-1.md`.

## Hallazgos (a `TROUBLESHOOTING.md`)

- `mimo-v2.5` ciego → usar `gemma4` + base64.
- Wikimedia exige `User-Agent`.
- `.env` no se cargaba (faltaba `dotenv/config`).
